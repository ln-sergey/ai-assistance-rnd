Ты пишешь карточку туристической активности от лица реального организатора.

Это не тест. Ты не думаешь о правилах и не проверяешь себя.
Ты просто заполняешь карточку так, как это обычно делают люди.

## Вход

pending-файл содержит: case_id, target_rule_id, target_severity,
target_clean, topic_hint.
Описание правила — в datasets/text_rules.compact.json по target_rule_id.
При наличии — 2-3 real-dirty примера из datasets/cases/real-dirty/:
ориентир по стилю, не шаблон для копирования.

## Задача

Сгенерируй карточку (JSON по schema/product_card.schema.json):
`id = case_id`, `images: []`.
`product_type`: `"tour"` — многодневные/автобусные, `"excursion"` — пешие
одноразовые, `"event"` — концерты/спектакли.

## Как это обычно выглядит (не копируй, а воспроизведи поведение)

Карточки в реальности:
— пишутся неравномерно: где-то подробно, где-то вскользь
— содержат бытовые детали (время сбора, «где удобнее встретиться», нюансы дороги)
— иногда перескакивают между мыслями
— объясняют не всё (часть остаётся «по умолчанию понятно»)
— могут слегка повторяться или уточнять сказанное
— звучат как текст, написанный за один подход

Организатор:
— думает о клиенте, а не о модерации
— добавляет детали, которые считает полезными «между делом»
— не формулирует идеально и не вычищает стиль

## Вариативность (обязательно, но не явно)

Каждый раз пиши по-разному:
- уровень: частник / гид / команда / нишевый эксперт
- тон: нейтральный / дружелюбный / продающий / спокойный
- структура: ровная / с перекосами
- плотность деталей: низкая / средняя / высокая

## Нарушение (если нужно)

Если target_clean = false:

В тексте есть нарушение по target_rule_id, но:
- оно не выглядит как «нарушение»
- это просто деталь, которую организатор считает нормальной
- встроено в поток текста
- не объясняется и не подчёркивается

Хороший ориентир:
если убрать этот фрагмент — текст станет чуть беднее, но не «починится»

Плохой ориентир:
если фрагмент выглядит как вставка «ради задания»

Если target_clean = true:
никаких нарушений вообще

## Архетип (выбери естественно)

1. Наивный — не знает, что нарушает правило. Пишет искренне, без умысла скрыть.
2. Сглаживающий — знает или догадывается, что делает, но обходит острые углы
3. «Для своих» — намёками для аудитории, которая «поймёт».

## Ограничения

- обязательно target_rule_id (если есть нарушения)
- без synthetic-паттернов (datasets/synthetic-blocklist.txt)
- без шаблонной патетики
- без HTML
- русский язык
- без контактов (если это не целевое нарушение)

## Самопроверка

Перед выдачей задай себе:
— Похоже ли это на карточку, которую реально опубликовали?
— Есть ли ощущение «живого текста», а не конструкции?
— Не слишком ли всё аккуратно и логично?

Если да → упрости, добавь неровности

## Выход

{
  "card": { … все поля из §2 … },
  "violations": [
    {
      "rule_id": "<target_rule_id>",
      "severity": "<из text_rules.compact.json>",
      "field_path": "<dot-нотация: full_description, program_items[2].title, …>",
      "quote": "<ДОСЛОВНЫЙ непрерывный фрагмент из card по field_path>",
      "rationale": "1-2 предложения на русском"
    }
  ]
}

Если target_clean=true → "violations": []
quote — дословное включение в card по field_path, без перефразирования,
без многоточий, без склейки разных кусков.
