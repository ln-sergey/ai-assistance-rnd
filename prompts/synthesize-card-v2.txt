# synthesize-card-v2
# Канонический промпт для синтеза product_card-карточки с одним
# целевым нарушением. Ведущая установка — карточка должна выглядеть
# как реальная попытка продавца протащить нарушение по target_rule_id,
# а не «учебный пример нарушения». Anti-clickbait критерии — в теле.
#
# Связанные артефакты:
#   - datasets/text_rules.compact.json (35 TXT-правил, severity, desc)
#   - datasets/schema/product_card.schema.json (целевая схема)
#   - datasets/synthetic-blocklist.txt (чёрный список фраз; этап 4)
#   - prompts/annotate-conservative-v1.txt (для blind re-annotation
#     результата в локальной сессии — Р8)
#   - docs/synthetic-guide.md (workflow + критерии натуральности;
#     pending, создаётся в этапе 6)
#   - docs/synthesize-subagent-template.md (шаблон делегации субагенту)
#   - docs/tz-synthetic-cards.md (контекст спринта P5)
#
# Применяется через Claude Code в локальной сессии. API-вызовы к
# целевым провайдерам и эталонным AI для подготовки тестовых данных
# запрещены — см. hard rule в AGENTS.md / docs/tz-synthetic-cards.md.
#
# История версий:
#   v1 — первая итерация. «Незаметная попытка протащить нарушение»
#        как ведущая установка. Список 12 разрешённых тематик, длины
#        полей в диапазоне 5-95 перцентиля real-карточек, чёрный
#        список маркеров синтетики, запрет на «нечаянные» нарушения.
#   v2 — fix фактической ошибки в идентификации правил:
#        - превосходные степени без подтверждения («лучший»,
#          «единственный», «самый известный») — это TXT-33
#          (Неподтверждённые заявления о статусе/качестве), не TXT-19;
#        - TXT-19 — это «Сторонние ссылки и контакты» (упоминания
#          телеграма, домена-маркетплейса в тексте карточки и т.п.).
#        Это уточнение влияет на список «легко скатиться» в разделе 6
#        и на пометку blocklist'а в разделе 4.

Ты пишешь карточку активности для маркетплейса туристических
активностей от лица продавца (организатора). Цель — карточка должна
выглядеть как реальная заявка, в которую продавец *незаметно* вплёл
одно нарушение по правилу target_rule_id (см. вход).

Главный критерий успеха: если бы карточку увидел модератор в общей
очереди, он бы прошёл мимо или отклонил только после внимательного
чтения. Если нарушение «бросается в глаза» по форме (CAPS LOCK,
восклицательные знаки, рекламная патетика, стандартные SEO-маркеры
синтетики) — карточка ИСКУССТВЕННО завышает recall модели и не годится
как ground truth для бенчмарка.

## 1. Вход

В контексте задачи доступны:

- target_rule_id (например, TXT-05) и его запись из
  datasets/text_rules.compact.json (id, severity, title, desc).
- target_clean (true | false). Если false — карточка обязана нарушать
  ровно target_rule_id и больше никаких правил из TXT-01..TXT-35. Если
  true — карточка чистая (контрольный clean-кейс из synthetic-clean/),
  violations возвращается пустым массивом.
- topic_hint — подсказка тематики из разрешённого списка (см. п. 3).
  Если отсутствует — выбираешь самостоятельно.
- case_id (например, synth_txt05_001) — подставишь в поле card.id.
- При наличии — 2-3 реальных примера нарушений по этому правилу из
  datasets/cases/real-dirty/. Использовать как ориентир по стилю; НЕ
  копировать формулировки.

## 2. Жёсткие требования к формату карточки

Карточка должна валидироваться по datasets/schema/product_card.schema.json.
additionalProperties: false на верхнем уровне — лишние поля не
допускаются. Все required-поля присутствуют:

- id — строка, минимум 1 символ. Подставить значение case_id из
  pending-файла (например, "synth_txt05_001").
- product_type — "tour" | "excursion" | "event". Выбрать по тематике
  (event для спектаклей/концертов, excursion для одноразовых пеших,
  tour для многодневных и автобусных).
- title — название (30-80 символов). Конкретное, без рекламной патетики.
- short_description — краткое описание для превью (80-200 символов,
  обычно 1-2 предложения).
- full_description — полное описание (400-1500 символов). Не дотягивай
  искусственно до верхней границы.
- program_items — массив {order, title, description}. 1-8 пунктов; для
  event допустим пустой массив. order — последовательно с 1.
- services — массив {name, description}. 0-6 элементов. Может быть [].
- location — {address, route_comment, meeting_point_comment}, все
  строки. Пустые строки допустимы; для event достаточно address.
- contacts_block — {public_comment}, строка. Обычно пустая или
  короткая заметка типа "Сбор за 10 минут до начала". ВНИМАНИЕ:
  телефоны, e-mail, telegram и ссылки на личные профили сюда не
  попадают — кроме случая, когда target_rule_id = TXT-07 (публикация
  персональных данных) или target_rule_id = TXT-19 (сторонние ссылки
  и контакты).
- schedule — {format, dates?, duration_minutes?}. format:
  "once"|"recurring"|"ondemand". duration_minutes — реалистично
  (60-720). dates — YYYY-MM-DD; для ondemand обычно отсутствует.
- images — всегда []. Синтетика без фото — out of scope спринта.
- age_restriction — строка вида "0+", "6+", "12+", "16+", "18+" или
  null. Для event обычно есть; для туров — реже.
- group_size — {min, max} (целые ≥ 1) или null. Реалистично:
  min 1-4, max 6-30.
- languages — массив ISO-639-1 кодов: ["ru"], ["ru", "en"], ["en"].

Стиль текста — нейтрально-деловой или дружелюбно-описательный, без
рекламной патетики. Конкретика: имена локаций, длительности, бытовые
детали программы, реалистичные ограничения.

## 3. Тематика (разрешённый список)

Выбирай topic_hint из 12 тематик: пешая экскурсия,
гастрономический тур, тур на природу/в горы, водная экскурсия,
автобусный тур, мастер-класс, спектакль/концерт (event), корпоративный
тимбилдинг, экскурсия с гидом-экспертом, тур по индустриальным/
промышленным объектам, детская экскурсия, исторический квест.

В одной партии генерации не более 1/3 карточек на одной тематике
(контролируется в synth:validate, этап 4).

## 4. Anti-clickbait — список запрещённого

1. Запрещённые фразы (точное вхождение, регистронезависимо):
   «лучший в мире», «лучший в России», «лучший в <город>»,
   «уникальная экскурсия», «не имеющий аналогов», «100 % гарантия»,
   «эксклюзивная программа», «секретный маршрут», «только у нас»,
   «непревзойдённый», «легендарный». Полный список —
   datasets/synthetic-blocklist.txt.
2. CAPS LOCK > 5 % символов в любом поле — запрещён. Исключение:
   target_rule_id = TXT-28 (избыточный капслок — само правило).
3. Эмодзи в title — запрещены полностью. В short_description /
   full_description — не более 2 эмодзи на 1000 символов. Исключение:
   target_rule_id = TXT-29 (избыточные эмодзи — само правило).
4. Повторяющиеся восклицательные знаки («!!», «!!!») — запрещены.
5. Превосходные степени без подтверждения («лучший», «единственный»,
   «самый известный» без источника) — это паттерн TXT-33
   («Неподтверждённые заявления о статусе/качестве»). Не вставляй
   случайно: либо target_rule_id = TXT-33 и так задумано, либо
   правило другое и таких фраз быть не должно.

## 5. Само нарушение должно быть органично вплетено

1. Не выделяй нарушение отдельным абзацем или маркированным пунктом.
   Оно встроено в осмысленный текст как естественная фраза
   организатора.
2. Не повторяй формулировки из desc правила и из реальных примеров —
   они даны как ориентир, не как шаблон.
3. Карточка остаётся валидной по схеме product_card даже после
   удаления нарушающего фрагмента (нарушение — добавление, не основа
   смысла карточки).
4. Длина и стиль нарушающего фрагмента соответствуют окружающему
   тексту. Если карточка нейтрально-описательная, нарушение тоже
   подаётся ровным тоном, без эмоционального нажима.

## 6. Запрет на «нечаянные» нарушения

При target_clean=false карточка обязана нарушать РОВНО target_rule_id
и больше никаких правил TXT-01..TXT-35. Особенно легко скатиться в:

- TXT-33 (превосходные степени без подтверждения, рекламная патетика)
  — самое частое загрязнение синтетики; следи за словами «лучший»,
  «эксклюзивный», «уникальный», «легендарный».
- TXT-19 (сторонние ссылки и контакты) — не упоминай в тексте
  телеграм-каналы, инстаграм, домены вроде «trip.ru», «booking»,
  «whatsapp» и подобные, кроме случая target_rule_id = TXT-19.
- TXT-25 (шаблонный текст без конкретики) — «провести незабываемое
  время», «прекрасный отдых», «отличная атмосфера» дают TXT-25 как
  побочное.
- TXT-26 (нечитаемый текст / технический мусор) — не пиши <p>, <br/>,
  &nbsp;, &laquo;, &mdash;.
- TXT-27 (смысл неясен) — описание должно быть конкретным.
- TXT-30 (неподходящий язык публикации) — пиши на русском, кроме
  случая target_rule_id = TXT-30.

Перед выдачей результата перечитай карточку и проверь, что ни одно
другое правило не сработало. Если уехал — переписывай тот фрагмент.

При target_clean=true проверка строже: карточка не нарушает НИ ОДНОГО
правила, всё то же anti-clickbait применяется без исключений.

## 7. Что вернуть

Один JSON-объект ровно такой формы (без обёрток ```json):

{
  "card": { /* полный product_card по схеме, со всеми required-полями
              включая id из case_id */ },
  "violations": [
    {
      "rule_id": "<target_rule_id>",
      "severity": "<точно как в text_rules.compact.json>",
      "field_path": "<dot-нотация: full_description, program_items[2].title, contacts_block.public_comment, services[0].description, …>",
      "quote": "<ДОСЛОВНЫЙ непрерывный фрагмент из card по field_path>",
      "rationale": "1-2 предложения по-русски, почему именно это нарушение"
    }
  ]
}

Если target_clean = true — violations: [].

Quote верифицируется (как в annotate-conservative-v1.txt) дословным
включением в card по field_path. Без перефразирования, без многоточий,
без склейки разных кусков, минимум 1 символ. Если по field_path поле
пустое — это поле не может быть field_path.
